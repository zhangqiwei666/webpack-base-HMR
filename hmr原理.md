# HMR（热模块替换）与更新编译原理

## HMR 基本原理
1. 开发服务器（如 webpack-dev-server）监听源代码变化。
2. 代码变动时，只重新编译变更的模块，不刷新整个页面。
3. 通过 WebSocket 通知浏览器有模块更新，浏览器接收新模块代码。
4. HMR 运行时将新模块“热插拔”到应用中，替换旧模块，保留应用状态。
5. 若模块无法热替换，则回退为整页刷新。

## 代码变动检测与编译
- 使用 Node.js 的 fs.watch、chokidar 等库监听文件变化。
- 检测到文件变动后，自动触发对应模块的重新编译（只编译变动部分）。
- 编译完成后，通过 HMR 通道将更新推送到浏览器。

## WebSocket 通知更新原理
1. 开发服务器启动 WebSocket 服务。
2. 浏览器端 HMR 客户端与服务器建立 WebSocket 长连接。
3. 代码变动并重新编译后，服务器通过 WebSocket 向客户端推送“更新消息”。
4. 浏览器收到消息后，拉取新模块代码并执行热替换，无需刷新页面。

## 总结
HMR 通过文件监听、增量编译和 WebSocket 实时推送，实现了高效的前端开发体验。
